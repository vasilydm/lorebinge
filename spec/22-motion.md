## 1. Принципы

1. **Тишина и удар (гибрид-темп).** Весь функциональный UI — быстрый и бесшумный (120–220 мс, `ease-standard`): навигация, кнопки, списки никогда не заставляют ждать. Вся драма сосредоточена в 4–5 **геро-моментах** (reward, level-up, флип карты, смерть стрика, splash) — там 600+ мс, `ease-emphasized`, Rive. Роскошь создаётся контрастом темпа, а не общей медлительностью.
2. **Кровь — событие, не фон.** Кровавые эффекты подчиняются правилу 70/20/10 (brand §4.1) и во времени: чем реже эффект, тем он дороже. Blood-wipe — только ритуальные пороги; сплэш — только ошибка; потёк — только смерть стрика.
3. **Движение — это свет и жидкость, не резина.** Характер бренда несут свечение (underglow, шиммер, ember) и течение (потёк, налив, wipe). Никаких bounce/jelly/конфетти — это язык Duolingo, не наш. Overshoot допустим только лёгкий (`ease-emphasized`) и только на наградах.
4. **Ноль хардкода.** Все длительности/изинги — токены из раздела 2 (`core/theme`), анимации — на UI-треде (Reanimated worklets / Rive render thread).

## 2. Токены (расширение brand §7)

| Токен | Значение | Где |
|---|---|---|
| `dur-fast` | 120 ms | тап-фидбек, переключатели, иконка таба |
| `dur-base` | 220 ms | переходы карточек, плашки, тосты, вердикт-баннер |
| `dur-slow` | 360 ms | модалки, шиты, открытие урока |
| `dur-drama` | 600 ms | геро-моменты: wipe, флип, потёк, blood-bloom |
| `dur-count` | 800 ms | count-up/down счётчиков (алмазы) |
| `ease-standard` | `cubic-bezier(0.2, 0, 0, 1)` | большинство |
| `ease-emphasized` | `cubic-bezier(0.2, 0, 0, 1.2)` | награды, акценты (лёгкий overshoot) |
| `ease-exit` | `cubic-bezier(0.4, 0, 1, 1)` | уходящие элементы (быстро прочь) |
| `loop-heartbeat` | цикл 1.2 s (двойной пик) | пульс активной ноды ([course-path.md §4](36-course-path.md)) |
| `loop-ember` | цикл ~3.5 s | «тление» battery-low, idle-шиммер стрика |
| `lock-reward` | 450 ms | окно неактивности перед dismiss наград (ревью #14) |

## 3. Базовый отклик (весь UI)

- **Кнопка primary (Continue / Done / Get the lore):** press-in — scale 0.97 + blood-underglow (тень `primary`, +20% opacity), `dur-fast`; release — обратно `ease-standard`. **Переход disabled→enabled** (когда задание выполнено и «Done» оживает): crossfade цвета + один проход ледяного шиммера по кнопке (`dur-base`) — кнопка «просыпается», взгляд падает на неё сам.
- **Текстовые/вторичные кнопки, табы:** opacity 0.6 на press, `dur-fast`. Иконка активного таба — micro scale-pop 1.0→1.12→1.0 (`dur-fast`), без подпрыгиваний.
- **Карточки (Home/Grid/Catalog):** press — lift: scale 0.98 → 1.0 c лёгкой тенью, `dur-fast`. Карточки Course Path — свой набор ([course-path.md](36-course-path.md)).
- **Появление списков:** без каскадных stagger-парадов; контент появляется сразу (fade `dur-fast`). Скелетоны — тёмный шиммер `surface`→`surface-2` (никогда светлый), период ~1.6 s.
- **Тосты/плашки:** въезд сверху `dur-base` `ease-standard`, уход `ease-exit`.

## 4. Переходы между экранами

- **Обычная навигация (push/pop в стеках, табы)** — системная и быстрая: нативный slide для push/pop, табы переключаются мгновенно (анимируется только иконка). Крови нет — навигация не событие.
- **Blood-wipe (`dur-drama`, полноэкранный стилизованный чернильно-кровавый wipe/растворение) — только ритуальные пороги:**
  1. **Вход в урок** (`course_path` → `lesson`): тьма «съедает» экран сверху вниз → раннер. Порог: ты входишь в историю.
  2. **Урок → Reward:** wipe → сцена награды (см. хореографию, §6.1).
  3. **Level-up** — blood-bloom за маскотом (brand §7).
  4. **Splash** — портрет Распутина проявляется из тьмы (не wipe, а проявление; та же семья).
  5. **Онбординг** — переходы между интро-экранами ([onboarding.md §4](35-onboarding.md)).
- **Paywall — намеренно БЕЗ драмы.** Появление — обычный slide-up шит `dur-slow`. Упор в стену — не праздник и не спектакль; холодная, быстрая, деловая плашка («The battery’s dry»). Не награждаем фрустрацию анимацией.
- **Reduce-motion:** wipe → простой fade-through-dark `dur-base`.

## 5. HUD: четыре стихии

### 5.1 💎 Алмазы — count-up
- **Начисление** (возврат с reward / докрутка на самом reward): число бежит вверх `dur-count` `ease-standard` (замедляясь к концу), по иконке — один проход **ледяного шиммера** (блик `ice`), лёгкий haptic light на финальном значении. Большие суммы не растягиваем: длительность фикс., меняется шаг счёта.
- **Списание** (автозаморозка, §5.3-тост): count-down `dur-count`, иконка на полсекунды охлаждается в `ice` (заморозка = лёд съел алмазы).

### 5.2 🔋 Батарея — молчаливый тик (решение)
- **Трата за ответ:** число просто сменяется с fade `dur-fast`. Без вспышек, капель и тряски — внимание пользователя в этот момент принадлежит заданию, не HUD.
- **Ценность заряда проживается на уровне состояний, не каждого тика:** при входе в `battery-mid` (янтарь) — однократный crossfade цвета `dur-base`; при входе в `battery-low` (≤3, кровь) — иконка начинает медленно «тлеть» (`loop-ember`: ±15% яркости, период ~3.5 s) — единственная цикл-анимация HUD, тихая тревога на периферии зрения.
- **Восстановление** (refresh при заходе): если заряд вырос — короткий count-up `dur-base` без шиммера (рутина, не награда).

### 5.3 🔥 Стрик
- **Инкремент** (первый урок за день; узнаём из `complete_lesson`): момент показывается **на экране Reward** (см. §6.1) — ember-вспышка огня + число +1 `ease-emphasized`. В HUD по возврате — число уже новое, без повторного спектакля.
- **Автозаморозка сработала (решение — тост-момент):** при заходе, поверх первого экрана — неблокирующий тост: иконка **замёрзшего огня** (лёд), текст в духе *“The ice held your fire. −300 💎”* (реплики — финализировать), `dur-base` въезд, виден ~3.5 s. Синхронно в HUD — count-down алмазов (§5.1). Трата прожита, путь к контенту не заблокирован.
- **Смерть стрика (решение — HUD-потёк, без интерстишла):** при первом показе HUD после захода с разрывом: число стрика (старое) окрашивается в `blood-bright` → **«истекает» вниз** (маска-потёк, `dur-drama`) → на его месте `0`, иконка огня гаснет в `muted` (crossfade). Одновременно — неблокирующий тост с канон-репликой *“Your streak bled out.”* (brand §1). Один раз на разрыв (флаг по `streak_broken_at`), haptic warning.
- **Тап по стрику** — открывает дропдаун (current/longest, §12.0); потёк и тосты этого не требуют.

### 5.4 Уровень
- Меняется редко (раз в 5 курсов) — вся драма в окне level-up (blood-bloom; **Rive — пост-MVP**, в MVP статичный портрет, §12.6). В HUD после закрытия окна — число сменяется с маленьким scale-pop `dur-fast`. 

## 6. Урок: прогресс, вердикты, награда

### 6.1 Хореография Reward (геро-момент №1)
`complete_lesson` → blood-wipe (`dur-drama`) → сцена: маскот (**MVP — статичный портрет**; Rive celebrate/encourage по `errors_count` — **пост-MVP**, §12.6) → **count-up алмазов** (`dur-count` + ледяной шиммер, haptic light) → если стрик инкрементнулся — **ember-вспышка** «Day N» (`dur-base`, `ease-emphasized`). Тапы игнорируются первые `lock-reward` 450 ms **после последнего бита** (ревью #14), затем — tap-to-dismiss.

### 6.2 Прогресс-бар урока
- Заполнение **по заданиям**: каждый Continue/Done двигает полосу на 1/N — `dur-base`, `ease-standard` (быстрый старт, мягкий доезд; не линейно). Цвет — `primary` (blood).
- **Фронт заполнения** — чуть более яркая «голова» (`blood-bright`, 2–3 px): полоса не растягивается, а **наливается**.
- **Последний сегмент** (урок завершён): доезд `ease-emphasized` с микро-overshoot + короткая вспышка всей полосы → сразу wipe в reward. Финал полосы = начало праздника.

### 6.3 Вердикт «верно»
- Баннер `success` въезжает снизу `dur-base`; иконка ✓ — быстрый pop (scale 0→1.1→1.0, `ease-emphasized`). Haptic success. Цвет **всегда** + иконка (colorblind, brand §9). Тихо и приятно — верных ответов много, они не должны утомлять.

### 6.4 Вердикт «неверно» — эскалация (решение)
- **1-я ошибка (или после верного ответа):** локально — **стилизованный blood-splash** в зоне баннера вердикта (`dur-base`) + shake выбранного варианта (3 колебания, ~300 ms) + ✗-pop. Haptic error.
- **2+ ошибки подряд (в рамках урока, счётчик сбрасывается верным ответом):** к локальному сплэшу добавляется **кровавая виньетка** по краям экрана — появляется `dur-base`, рассасывается ~800 ms (как индикация урона в играх). Драма растёт с провалом: «здоровье уходит».
- В matching/timeline вердикт-на-месте — тоже цвет + ✓/✗ + shake (ревью #13), без виньетки (она глобальная, по счётчику подряд).
- **Reduce-motion:** без shake и виньетки — только цвет + иконка.

### 6.5 Course Path (ссылки на решения)
- Пульс активной карты scale+underglow в ритме `loop-heartbeat` — [course-path.md §4](36-course-path.md).
- Флип разблокировки `path-flip` = `dur-drama` `ease-emphasized` — [course-path.md §3](36-course-path.md).
- Тап по карте — lift + усиление свечения `dur-fast` → переход-порог в урок (§4).

## 7. Haptics (пары к движению)

Дополняет таблицу хаптики §12.x: каждый haptic идёт **в момент пика** анимации, не раньше.

| Событие | Haptic |
|---|---|
| Вердикт верно | `success` (notification) |
| Вердикт неверно | `error` + лёгкий `impact` на пике shake |
| Count-up алмазов (финал) | `light` |
| Флип карты (раскрытие) | `medium` |
| Смерть стрика (потёк дошёл до 0) | `warning` |
| Кнопка primary press | `selection` (iOS) / none (Android, чтобы не дребезжало) |
| Reward dismiss | нет (тап и так получает визуальный ответ) |

## 8. Reduce-motion и перфоманс

- **`prefers-reduced-motion`:** все циклы выключены (пульс ноды, тление батареи, шиммеры), wipe→fade, shake/виньетка→статичный цвет+иконка, count-up→мгновенное значение с fade, маскот — статичный кадр. Сценарная информация (вердикт, потёк-смысл «стрик умер») передаётся всегда — анимация лишь усиливает.
- **Бюджет циклов: ≤2 одновременно на экран** (напр. пульс активной ноды + тление battery-low), приоритет у контентного. Все loops — Reanimated worklets/Rive, ничего на JS-треде; при скролле списков циклы не приостанавливаем (они на UI-треде), но новые не запускаем.
- **Инструменты:** Reanimated — трансформы/opacity/count-up + аним. SVG-батарея (§12.0); **react-native-svg** — гемификация-иконки; **Skia** — underglow, шиммеры, виньетка (шейдер); Lottie — только если ассет дешевле сделать в AE (потёк, splash). **Rive (маскот, blood-wipe/bloom) — пост-MVP** (в MVP маскот статичный, §12.6). Прероллить wipe-ассет в память до первого порога (чтобы первый вход в урок не дёрнулся).
